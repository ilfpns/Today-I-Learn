> **Driver**
> 

---

⇒ OS와 디바이스를 이어주는 소프트웨어

**ㅤ**

- 레지스터 매핑 읽기/쓰기
- 인터럽트 처리
- DMA 설정
- 디바이스 초기화
- 디바이스 전원 관리

디바이스(하드웨어)를 제어하기 위해 커널이 제공하는 소프트웨어가 드라이버이다

**ㅤ**

> Device Driver
> 

---

⇒ 하드웨어를 커널에서 제어할 수 있도록 만들어진 인터페이스

**ㅤㅤㅤㅤ**

1. Character Device
    - device를 파일처럼 직접 접근하여 직접 read/write 수행

**ㅤ**

1. Block Device 
    - file system을 기반으로 block단위로 데이터를 read/write 수행
    - 일전한 크기의 배수로만 읽고 쓸 수 있다 (512, 1024, …)

**ㅤ**

1. Network Device 
    - network의 물리 계층과 frame 단위의 데이터 송수신
    - BSD 소켓 인터페이스로 하위 네트워크 계층에 접근할 수 있다

**ㅤ**

**특징**

- 잘못되면 시스템에 큰 영향을 끼침
- 커널이나 자신이 속한 서브시스템에 표준 인터페이스를 제공
- 메모리 할당, 인터럽트 전달, 대기 큐 같은 커널 서비스 사용
- 부팅 후 디바이스 드라이버가 초기화 될 때, 시스템은 제어할 하드웨어 장치를 가지며, 없다 하더라도 문제가 되지 않음
