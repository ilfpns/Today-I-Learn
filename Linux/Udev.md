> **Udev**
> 

---

⇒ 리눅스 커널 장치 관리자이다

⇒ devfsd/hotplug를 계승하는 udev는 주로 /dev  디렉터리의 장치 노드를 관리한다

**ㅤ**

- Udev Rule을 사용하기 위해선 udevadm 명령을 통해 장치의 상세 정보를 확인한다

**ㅤ**

**/dev**   

- 하드웨어를 디바이스 노드로  표현한 디렉토리
- X server가 /dev/input/mice 같은 노드를 통해 마우스 이벤트를 읽음

**ㅤ**

> **왜 udev인가**
> 

---

- 커널이 준 이름 대신 다른 이름의 디바이스 노드를 생성할 수 있다
- 외부 프로그램 결과를 이용해 이름/링크 설정
- 네트워크 인터페이스 이름 재정의

**ㅤ**

> **실행**
> 

---

⇒ /etc /udev/rules.d/

**ㅤ**

- 파일 확장자 : *.rules
    
    ⇒ 한 줄은 한 규칙으로 인식한다, 여러 줄로 나뉘면 다른 규칙으로 취급된다
    

**실행이 필요한 상황**

| 작업 | udev 필요 여부 |
| --- | --- |
| GPIO 제어 | ❌ 필요 없음 |
| I2C/SPI/UART (직접 핀 연결) | ❌ 필요 없음 |
| USB 기반 센서/기기 관리 | ✔ 반드시 필요 |
| USB 장치 자동 실행(백업/마운트) | ✔ 반드시 필요 |
| 여러 USB 장치 이름 고정 | ✔ 반드시 필요 |
