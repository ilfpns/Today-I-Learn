> **MMU**
> 

---

⇒ 가상주로를 물리주소로 변환하는 하드웨어 블록

- 프로세서가 메모리에 접근할 때 mmu에 가상주소를 전달하고, mmu는 변환테이블을 참고해 실제 물리주소로 변환해 전달

**ㅤ**

**부팅시**

1. 리눅스는 메모리 일부분을 MMU테이블에 할당
2. 관리 정보를 MMU 테일에 기록
3. 기록이 끝난 MMU테이블의 메모리 위치를 MMU에 알려주고 작동

**ㅤ**

**왜 MMU를 사용할까**

1. 프로세스 끼리의 간섭을 막음
    
    ⇒ 어디에 로드될지 모르는 프로세스들의 충돌을 막음
    

**ㅤ**

1. 주소를 재배치하고 보호함

**ㅤ**

> **MMU의 작동 프로세스**
> 

---

- CPU : 가짜 주소만을 봄
- MMU : 가짜 주소 ↔ 진짜 주소 변환
- RAM : 진짜 주소만을 봄

**ㅤ**

```powershell
CPU : 0x0001 요청
MMU : 0x0001을 받음, RAM에게 0x0001ABBC 송신
RAM : 0x0001ABBC 처리
```

**ㅤ**

> **ioremap**
> 

---

⇒ IO 주소 공간을 가상 주소 공간에 할당한다

**ㅤ**

```c
#include <asm/io.h>

void* ioremap(unsigned long offset, unsigned long size)
```

파라미터 : offset 물리적 I/O 주소, Size할당하고자 하는 I/O 영역의 크기
