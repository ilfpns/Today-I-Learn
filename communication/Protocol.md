# Serial Communication

직렬 연결

→ 한 번에 하나의 비트단위로 1과 0의 신호로 데이터를 전송하는 통신 방법.

    동기식과 비동기식으로 나뉜다.

| 동기식 통신 | 비동기식 통신 |
| --- | --- |
| 데이터 신호와는 별도로 동기신호를 함께 보낸다 | 데이터 신호만을 보내고 각각의 방식에 따라 데이터 비트를 찾아낸다 |
| 언제 데이터를 주고 받을지 정해져 있다 | 언제 주고받을지 정해져 있지 않기에 방식을 맞춰야 한다 |

### 비동기

→ 비동기식 통신에서 주는 쪽과 받는 쪽이 규약을 맞추지 않는다면 데이터를 못 찾을 수 있다

시리얼통신 동기 규약

---

통신 속도(Baud rate)

→ 초당 몇개의 비트를 읽을 것인지에 대한 규약 

→ 1200, 2400, 4800, 9600 등의 속도를 선택할 수 있다

데이터 비트(Data bit)

→ 실제 주거나 받을 데이터에 대한 비트 영역 규약

→ 5 ~ 9 비트를 선택할 수 있다

패리티 비트(Pairty bit)

→ 데이터 내의 1비트 수를 체크하여 짝수와 홀수에 따라 에러를 검출하는 방법

스탑 비트(Stop bit)

→ 주거나 받는 쪽의 통신을 언제 끊을 지에 대한 비트 영역이다

# Protocol

→ 메시지를 주고 받는 양식과 규칙의 체계이다. 즉 통신 규약 및 약속

**프로토콜 기본 요소**

- **구문(Syntax)** : 전송하고자 하는 **데이터의 형식(Format), 부호화(Coding), 신호 레벨(Signal Level)** 등을 규정
- **의미(Semantics)** : 두 기기 간의 효율적이고 정확한 정보 전송을 위한 협조 사항과 오류 관리를 위한 제어 정보를 규정
- **시간(Timing)** : 두 기기 간의 통신 속도, 메시지의 순서 제어 등을 규정
