⇒ HW구성을 SW에 설명하기 위해 사용되는 데이터 구조

⇒ ARM기반의 SoC에서 널리 사용된다

**ㅤㅤ**

⇒ 커널에게 하드웨어 정보를 알리기 위해 

**ㅤ**

**왜 생겨났나?**

- 하드웨어 구성은 C로 작성되지만, 하드웨어가 복잡해지며 유지보수가 어려워짐.
- 이는 하드웨어 정보를 독립적인 데이터 파일로 분리한 DTS로 해결할 수 있다.

**ㅤㅤ**

**주요 개념**

1. 하드웨어 추상화 : DTS는 하드웨어 구성 정보를 커널 코드와 분리하여 독립적으로 관리함
2. 유연성 : 동일한 커널 바이너리로 다양한 하드웨어 플랫폼을 지원함
3. 표준화 : 어느 기준에 종속되지 않는 표준 방식으로 하드웨어를 기술함

**ㅤㅤ**

**장점**

- 유지보수
- 코드 복잡성 감소
- HW간의 재사용성 향상

**ㅤㅤㅤ**

**구조**

- 노드와 속성으로 구성된다
    1. 노드 : 하드웨어 장치를 나타냄
    2. 속성 : 해당 장치의 설정값
- 확장자 : dts, dtsi

- reg : 장치의 메모리 매핑 주소와 크기
- interrupt : IRQ 번호와 플래그를 정의
- clocks : 필요한 클럭 소스를 정의

    - 트리 형태로 작성한다

    - 부트로더가 DTB를 커널에 전달
