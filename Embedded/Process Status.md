> Batch Job
> 

---

⇒ 사용자가 수정이나 데이터 변경을 할 수 없는, 모든 작업을 한번에 처리하는 시스템이다.

**ㅤ**

1. Create : 프로세스가 메모리에 올라와 있는 상태
2. Run : CPU를 얻어서 자신의 일을 수행하는 상태, 일괄 작업 시스템은 CPU를 혼자 점유하기에 효율이 떨어진다
3. Terminate : 주어진 작업을 완료하여 프로세스 컨트롤 블록을 제거하고 메인 메모리에서 할당을 놓아준 상태

**ㅤ**

> **시분할 시스템 상태**
> 

⇒ 여러 프로세스에 CPU를 함께 할당하여, 다수의 작업을 함께 사용한다

**ㅤ**

1. Creating : OS에 프로세스의 PCB가 생성되며, 프로세스가 메인 메모리에 적재되어 실행 준비를 마침
2. Readying : CPU를 얻을 때까지 기다리며, Ready Queue에 들어와 있는 상태이다
3. Runnig : Ready Queue에 있던 프로세스가 자신의 차례를 부여받아 실제로 수행되는 상태
4. Waiting : 어떤 인터럽트가 발생했을 때 이에 대한 대기를 하며, 이 때 프로세스는 잠시 CPU를 놓고 block 상태가 된다
5. Terminating : 작업을 모두 마치고 PCB를 제거 후 메인 메모리에서 빠져나간 상태이다
