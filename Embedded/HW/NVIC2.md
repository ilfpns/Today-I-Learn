> **Nested Vectoreed Interrupt Controller**
> 

---

⇒ 중첩 벡터형 인터럽트 제어기라는 뜻으로, 특수 PIC이다

⇒ 인터럽트를 받고 이를 처리함

**ㅤㅤ**

- 구조적으로 Cortex M4 프로세서와 밀접하게 연결되어, 빠른 인터럽트 처리가 가능하다
- 모든 인터럽트 예외에 대한 우선순위를 결정하고 처리한다

<img width="794" height="577" alt="image" src="https://github.com/user-attachments/assets/e76b32a4-9e8e-4042-832b-b750fd56109c" />

**ㅤ**

**특징**

- 82개의 마스크 가능한 인터럽트 채널
- Cortex M4 프로세서로부터 입력되는 16개의 인터럽트 채널
- 16단계 우선순위 설정 가능
- 우선순위 설정에 4비트를 이용
- 빠른 인터럽트 처리 시간

**ㅤ**

**처리 순서**

1. Stacking
    
    ⇒ 인터럽트 처리에 사용할 현재 레지스터의 주소를 스택에 저장한다
    
2. Vector Fetch
    
    ⇒ 벡터 테이블에서 인터럽트 핸들러의 시작 주소를 Fetch
    
3. Register Update
    
    ⇒ 인터럽트 핸들러로 진입할 때 관련된 레지스터가 업데이트한다
    
4. ISR
    
    ⇒ 발생된 인터럽트에 해당되는 ISR을  실행한다
    
    **ㅤ**
    

**중첩 인터럽트**

⇒ 현재 인터럽트보다 우선 순위가 높은 인터럽트가 발생할 떄, 우선 순위가 높은 인터럽트부터 처리하는 기능

⇒ 인터럽트 속의 인터럽트라 중첩 인터럽트임

**ㅤ**

**벡터형 인터럽트**

⇒ 시작 주소부터 순서대로 비교하면서, 일치하는 값이 있을 경우 읽어들이는 방식

⇒ 데이터 값의 주소를 위치와 방향으로 정리한 표인 벡터 테이블을 조회하며 값을 찾는다. 주소를 찾는다면 ISR을 실행할 것이다
